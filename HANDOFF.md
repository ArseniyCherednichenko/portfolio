# portfolio — Session Handoff

> Arseniy Cherednichenko's personal portfolio site. Living context for a fresh
> Claude session (or the daily routine). Goal: grow this into a polished,
> motion-led, professional portfolio over time, applying the craft from Guided.

## Stack
React 18 + Vite + TypeScript (strict) + **Tailwind v4** (via `@tailwindcss/vite`, classes auto-detected, no content config) + **Framer Motion** for animation + **react-router-dom v6** for client-side routing. Dark theme, lime accent `#DCF87C`.

## Typography
Two faces. **Inter** (system fallback stack) carries body copy and UI. **Fraunces** — a self-hosted variable serif (`public/fonts/fraunces-latin*.woff2`, opsz 9..144, wght 300..700) — is the display face for headlines, giving the site an editorial voice instead of a templated all-sans look. `@font-face` (latin + latin-ext subsets) and a Tailwind `@theme` token `--font-display: "Fraunces", ui-serif, Georgia, serif` live in `src/index.css`; that token generates the **`font-display`** utility, and a sibling `.font-display { font-optical-sizing: auto }` rule lets the optical-size axis track the rendered size. The latin woff2 is `<link rel="preload">`ed in `index.html` so headlines paint without a flash. Apply `font-display` to display-level headings only (hero/page titles, section H2/H3, project titles, modal titles, big statement lines, ProjectPoster name); keep small labels/eyebrows/sub-card headings in Inter for hierarchy. The variable subset is weight-capped at 700 (`font-bold`).

## Architecture (multi-page)

- `src/App.tsx` = route table only (`<Routes>`).
- `src/components/Layout.tsx` = persistent shell (Aurora + Nav + footer) with `<AnimatePresence>` page transitions (keyed on pathname), a `ScrollManager` that scrolls to `#hash` targets or resets to top on navigation, a site-wide `ScrollProgress` reading bar (top edge) and a `BackToTop` floating control.
- **Scroll orientation system**: `ScrollProgress` (springed lime reading bar pinned to the viewport top, `useScroll`+`useSpring`), `BackToTop` (floating bottom-right control that springs in past one screenful, smooth-scrolls home), both site-wide in `Layout`; `SectionNav` (`src/components/SectionNav.tsx`) is a `lg:`-only right rail used on Home — an `IntersectionObserver` scroll-spy over the `intro/about/range/process/work/toolkit/contact` section ids that lights the active marker (spring `layoutId` ring), labels reveal on hover, click glides to the section. All reduced-motion aware.

Currently expands the `WorkDetail` poster; ready for any future visual media.
- **Cursor** (`src/components/Cursor.tsx`): site-wide custom cursor mounted in `Layout`. A tight lime dot springs hard toward the pointer while a softer ring lags behind; over interactive elements (`a, button, [role=button], input, textarea, select, label, summary, [data-cursor]`) the ring swells and fills faintly lime and the dot fades out, and it shrinks on press. Activates ONLY on `(pointer: fine)` with motion allowed — touch and reduced-motion users keep the native cursor (it returns `null` otherwise). When active it adds `body.cursor-none` (`index.css`) to hide the native cursor; layers fade out when the pointer leaves the window. Add `data-cursor` to any element to trigger the hover state. Featured as a Playground experiment card.
- **ScrollVelocity** (`src/components/ScrollVelocity.tsx`): React Bits-style scroll-velocity text band. Takes `rows` (`{ text, baseVelocity?, className? }[]`); each row renders 4 copies of its text in a `motion.div` whose `x` is a `wrap`ped offset driven by an `useAnimationFrame` loop — a constant base drift that speeds up and flips direction with the page's scroll velocity (`useScroll`->`useVelocity`->`useSpring`->`useTransform`). Per-row `className` lets rows differ (used to pair a filled white row with a lime `-webkit-text-stroke` outlined row). Under reduced-motion it renders one static, non-clipped line per row. Used full-bleed on Home (statement band between Work and Toolkit, masked with edge gradients) and as a Playground experiment.
- **DotGrid** (`src/components/DotGrid.tsx`): self-contained interactive canvas dot grid. Builds an evenly-spaced grid (configurable `gap`/`dotSize`/`proximity`/`push`), each dot a tiny spring pushed away from the pointer and pulled back to its origin, brightening toward lime within `proximity`. DPR-aware, `ResizeObserver` rebuilds on resize, RAF loop cleaned up on unmount; under reduced-motion it draws a single static evenly-lit grid (no loop). Used as the full-width "Interactive field" showcase on the Playground; reusable as an ambient layer anywhere.
- **CardStack** (`src/components/CardStack.tsx`): React Bits-style auto-advancing 3D card deck. Takes `cards` (`StackCard[]` of `{ tag, title, body }`) + optional `interval`. The front card recedes to the back on a timer; cards behind peek above it (offset/scale/tilt for depth, computed by `pose(pos)`). Advances on click, on a progress-dot jump, or the timer; pauses on hover/focus; only the front card is focusable/interactive (`aria-hidden` + `pointer-events` gated). Under reduced motion it renders a calm static, fully-readable list with no timer. Used on Home's new **Range** section (`#range`, between About and Work) to frame Arseniy as a multi-disciplinary builder — frontend, native iOS, backend/data, applied AI, motion/design — actively de-centering Guided; also a Playground experiment. `Home.tsx`'s `SECTIONS`/`SectionNav` includes the `range` id.
- **ScrollReveal** (`src/components/ScrollReveal.tsx`): React Bits-style scroll-linked, word-by-word text reveal. Takes a `children` string (and optional `highlight` word list painted lime); splits it into words, and as the paragraph travels through the viewport each word's opacity (`0.12 -> 1`) and lift (`6 -> 0`) map to its own slice of the block's scroll progress (`useScroll` target offset `['start 0.85', 'end 0.45']` -> per-word `useTransform`), so the sentence assembles itself under the reader's scroll. Honest to assistive tech: the full string is the `aria-label` and each visual word is `aria-hidden`. Under reduced motion it renders the finished line statically. Used as a broad **"What I am about"** statement on the About page (between intro and Now — deliberately de-centering Guided by framing the craft that carries across all his work) and as a full-width Playground experiment.
- **DecryptedText** (`src/components/DecryptedText.tsx`): text that resolves from random glyphs into the final string left-to-right. Plays once on view (`useInView`) and replays on hover; honest `aria-label` carries the real text. Under reduced-motion it renders the final string immediately with no scrambling. Used in the Playground "Decrypt-on-view text" experiment.
- **MagnetLines** (`src/components/MagnetLines.tsx`): React Bits-style canvas grid of small lines ("needles"). Each needle eases its angle toward the pointer (shortest-arc lerp via `angleDelta`) and brightens/thickens toward lime within `proximity`; pure canvas + RAF for a dense grid, DPR-aware, `ResizeObserver` rebuilds on resize, loop cleaned up on unmount. Under reduced motion it draws a calm, static radial (every needle aimed at the grid centre) with no animation loop. Used as a live full-bleed backdrop behind the Home **closing contact band** (`#contact`, radial-masked, low opacity, `pointer-events-none` so it never blocks the CTA) and as a full-width **Playground** experiment ("Needle field"). Honest ambient art, reusable anywhere as an interactive layer.
- **VariableProximity** (`src/components/VariableProximity.tsx`): TextPressure-style display text that exploits the self-hosted **variable** Fraunces face. As the pointer nears each letter, that letter's `wght` axis swells from light toward bold and its `opsz` axis opens up (set via inline `font-variation-settings`), so the line appears to lean toward the cursor and eases back as it leaves. Pure `requestAnimationFrame` with two passes per frame (read all letter rects, then write all styles — no thrash) and **no per-letter React state**, so long lines stay smooth. Letters are grouped into per-word `nowrap` wrappers so lines break only between words. Honest to assistive tech: the container carries the real string as `aria-label`/`role="text"` and every visual letter is `aria-hidden`. Under reduced-motion it renders the finished line at a fixed weight with no listeners or loop. Used as the **About** closing CTA headline ("Want to see the work?", with a "move your cursor" prompt) and as a full-width **Playground** experiment ("Lean in. / Type that feels you.").
- **ScrollStack** (`src/components/ScrollStack.tsx`): Apple / React Bits-style scroll-driven stacking cards. Takes `cards` (`ScrollStackCard[]` of `{ tag, title, body }`). Each card is wrapped in a `position: sticky` box pinned at `TOP + i*PEEK` px from the viewport top (so the deck fans into a visible peek); a single section-level `useScroll` (`offset: ['start start','end end']`) drives every card's `scale`/`opacity` via `useTransform` — once the next card lands on top (~`(i+1)/n`) the covered card scales down (`-0.04` per card stacked above) and dims (`-0.12`), so the growing deck reads as depth. Opaque card backgrounds (`from-[#161616] to-[#0d0d0d]`) cleanly hide the cards beneath; no per-card scroll listeners. Sticky resolves to the viewport because the `Layout` page wrapper's transform settles to `none` after the route transition. Under reduced motion it renders a calm, fully readable vertical list with no pinning/scaling/listeners. Used for Home's new **Process** section (`#process`, between Range and Work) — an honest "How I build" five-step process (Understand / Shape / Build / Refine / Ship) that de-centers Guided by foregrounding the craft that carries across every project — and as a full-width **Playground** experiment. `Home.tsx`'s `SECTIONS`/`SectionNav` includes the `process` id.
- **TrueFocus** (`src/components/TrueFocus.tsx`): React Bits-style focus-frame word row. Takes `words` (string list) + optional `interval`/`blur`. A lime corner-bracket frame (a springed `motion.div` whose `x/y/width/height` track the measured rect of the focused word, inflated by a small pad) glides to the active word, which snaps to `blur(0)`/full opacity while the others sit blurred and dimmed; it auto-cycles on a timer and hovering a word pins the focus there (pauses the timer until the pointer leaves). Word rects are measured via `useLayoutEffect` + a `resize` listener relative to the container. The words stay real, selectable text (only the frame is `aria-hidden`), so it reads normally to assistive tech. Under reduced-motion it renders a calm, fully legible static line with no frame, blur, or timer. Used on Home's **Range** section (a disciplines line — Frontend / iOS / Backend / Applied AI / Motion — that draws the eye across his range, de-centering Guided) and as a **Playground** experiment ("True focus", cycling Design / Build / Refine / Ship).
- **BentoGrid** (`src/components/BentoGrid.tsx`): reusable bento layout. `BentoGrid` is a responsive CSS grid (`grid-cols-2` mobile, `sm:grid-cols-4` with `auto-rows-[150px]`) whose cells stagger in on scroll via Framer variants (`staggerChildren`, `whileInView`, once). `BentoCell` is the styled card: a lime radial **spotlight** (`useMotionTemplate`) tracks the cursor per cell, the border warms, and the cell lifts (`whileHover y:-4`); pass span utilities (e.g. `col-span-2 sm:row-span-2`) via `className`, and `interactive={false}` to disable the lift. Under reduced motion it drops all listeners/variants and renders calm static cards. Used by the About **"At a glance"** snapshot and as a full-width **Playground** experiment.
- **useBerlinTime** (`src/hooks/useBerlinTime.ts`): shared hook returning live `{ time, hour, awake }` for Europe/Berlin (1s `setInterval` + `Intl.DateTimeFormat`); `awake` is a soft 08:00–24:00 signal, never a hard promise. Single source of truth for the live Berlin clock, used by both the Contact page and the About snapshot (extracted from the old inline `Contact` copy so they never drift).
- **At a glance** (About `Snapshot`): a live, scannable bento snapshot between the "What I am about" statement and the Now block — a big **live Berlin clock** cell (`useBerlinTime`, pinging "around / probably asleep" status dot), a "Currently" one-liner, a disciplines/range chip cell (de-centering Guided), a "Shipped" `AnimatedCounter` of `CASE_STUDIES.length` linking to `/work`, and an open-source GitHub cell.
