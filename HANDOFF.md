# portfolio — Session Handoff

> Arseniy Cherednichenko's personal portfolio site. Living context for a fresh
> Claude session (or the daily routine). Goal: grow this into a polished,
> motion-led, professional portfolio over time, applying the craft from Guided.

## Stack
React 18 + Vite + TypeScript (strict) + **Tailwind v4** (via `@tailwindcss/vite`, classes auto-detected, no content config) + **Framer Motion** for animation + **react-router-dom v6** for client-side routing. Dark theme, lime accent `#DCF87C`.

## Typography
Two faces. **Inter** (system fallback stack) carries body copy and UI. **Fraunces** — a self-hosted variable serif (`public/fonts/fraunces-latin*.woff2`, opsz 9..144, wght 300..700) — is the display face for headlines, giving the site an editorial voice instead of a templated all-sans look. `@font-face` (latin + latin-ext subsets) and a Tailwind `@theme` token `--font-display: "Fraunces", ui-serif, Georgia, serif` live in `src/index.css`; that token generates the **`font-display`** utility, and a sibling `.font-display { font-optical-sizing: auto }` rule lets the optical-size axis track the rendered size. The latin woff2 is `<link rel="preload">`ed in `index.html` so headlines paint without a flash. Apply `font-display` to display-level headings only (hero/page titles, section H2/H3, project titles, modal titles, big statement lines, ProjectPoster name); keep small labels/eyebrows/sub-card headings in Inter for hierarchy. The variable subset is weight-capped at 700 (`font-bold`).

## Architecture (multi-page)
- `src/main.tsx` wraps `<App/>` in `<BrowserRouter>`.
- `src/App.tsx` = route table only (`<Routes>`).
- `src/components/Layout.tsx` = persistent shell (Aurora + Nav + footer) with `<AnimatePresence>` page transitions (keyed on pathname), a `ScrollManager` that scrolls to `#hash` targets or resets to top on navigation, a site-wide `ScrollProgress` reading bar (top edge), a `BackToTop` floating control, and a `Preloader` intro curtain mounted last (above the whole shell).
- **Preloader** (`src/components/Preloader.tsx`): a once-per-session intro curtain mounted at the top of `Layout` (`z-[200]`). On the first load of a tab it renders a dark overlay with the name, a `GradientText` counter that eases 0→100 off the rAF clock (`COUNT_MS`), and a filling lime line; when the count completes, a row of `PANELS` (6) vertical panels wipes upward in a staggered sequence (`i*0.06` delay) to reveal the site, then the component unmounts (`setShow(false)` on the last panel's `onAnimationComplete`, guarded by `leaving` so the initial mount never closes it early). Plays exactly once via a `sessionStorage` `intro-played` flag (set immediately on show, so a mid-intro navigation can't re-trigger it), and locks `body` scroll while up. Reduced-motion users are decided out **synchronously in the `useState` initializer** (`useReducedMotion()` read first), so they never see even a flash of the blocking overlay — the intro is skipped entirely. Honest brand motion, no content claims; `aria-hidden` and non-blocking to assistive tech (the real page is fully in the DOM underneath).
- **Scroll orientation system**: `ScrollProgress` (springed lime reading bar pinned to the viewport top, `useScroll`+`useSpring`), `BackToTop` (floating bottom-right control that springs in past one screenful, smooth-scrolls home), both site-wide in `Layout`; `SectionNav` (`src/components/SectionNav.tsx`) is a `lg:`-only right rail used on Home — an `IntersectionObserver` scroll-spy over the `intro/about/range/process/work/toolkit/contact` section ids that lights the active marker (spring `layoutId` ring), labels reveal on hover, click glides to the section. All reduced-motion aware.
- Pages live in `src/pages/`: `Home.tsx` (hero/about teaser/range/process/ethos/work/toolkit/explore/contact + project modal), `About.tsx` (intro, scroll-reveal statement, At-a-glance snapshot, Now block, How-I-work principles, Path timeline, a Questions `Accordion` FAQ, and an interactive `VariableProximity` cursor-pressure CTA headline), `Work.tsx` (`/work` index — animated ledger of all projects), `Playground.tsx` (live motion experiments gallery + full-width interactive `DotGrid` field), `WorkDetail.tsx` (`/work/:slug` project case study), `Contact.tsx` (`/contact` — deep-linkable contact page), `Colophon.tsx` (`/colophon` — how the site is built), `Resume.tsx` (`/resume` — one-page, print-ready CV), `NotFound.tsx` (404).

- **Cursor** (`src/components/Cursor.tsx`): site-wide custom cursor mounted in `Layout`. Takes `rows` (`{ text, baseVelocity?, className? }[]`); each row renders 4 copies of its text in a `motion.div` whose `x` is a `wrap`ped offset driven by an `useAnimationFrame` loop — a constant base drift that speeds up and flips direction with the page's scroll velocity (`useScroll`->`useVelocity`->`useSpring`->`useTransform`). Per-row `className` lets rows differ (used to pair a filled white row with a lime `-webkit-text-stroke` outlined row). Under reduced-motion it renders one static, non-clipped line per row. Used full-bleed on Home (statement band between Work and Toolkit, masked with edge gradients) and as a Playground experiment.

- **HorizontalScroll** (`src/components/HorizontalScroll.tsx`): a pinned horizontal-scroll section. Takes `panels` (`HPanel[]` of `{ tag, title, body }`) + optional `className`. While the section is in view it `sticky top-0`-pins a full-height window and a flex row of panels translates sideways, so vertical scroll maps to horizontal travel — the panels glide past as you scroll down. Travel is **measured** (a `ResizeObserver` reads the track `scrollWidth` minus the visible window `clientWidth`, recomputed on resize) rather than hard-coded percentages, so it adapts to panel count and screen size; the outer height is set to `viewportHeight + distance` so a section-level `useScroll` (`offset: ['start start','end end']`) maps progress 0->1 onto exactly that travel, and a `useSpring` smooths the `x`. Each panel is a gradient card with a number, lime eyebrow tag, Fraunces title and body. Under reduced motion it degrades to a plain native horizontal scroller with CSS snap points (no pinning, no listeners) — fully readable. Used on Home's **Range** section (`#range`, between About and Work) — a full-bleed glide through his disciplines (Frontend / iOS / Backend / Applied AI / Motion, with the `TrueFocus` line above) that frames Arseniy as a multi-disciplinary builder, actively de-centering Guided — and as a full-width **Playground** experiment. `Home.tsx`'s `SECTIONS`/`SectionNav` includes the `range` id.

- **VariableProximity** (`src/components/VariableProximity.tsx`): TextPressure-style display text that exploits the self-hosted **variable** Fraunces face. As the pointer nears each letter, that letter's `wght` axis swells from light toward bold and its `opsz` axis opens up (set via inline `font-variation-settings`), so the line appears to lean toward the cursor and eases back as it leaves. Pure `requestAnimationFrame` with two passes per frame (read all letter rects, then write all styles — no thrash) and **no per-letter React state**, so long lines stay smooth. Letters are grouped into per-word `nowrap` wrappers so lines break only between words. Honest to assistive tech: the container carries the real string as `aria-label`/`role="text"` and every visual letter is `aria-hidden`. Under reduced-motion it renders the finished line at a fixed weight with no listeners or loop. Used as the **About** closing CTA headline ("Want to see the work?", with a "move your cursor" prompt) and as a full-width **Playground** experiment ("Lean in. / Type that feels you.").

Under reduced-motion it draws one calm static set of waves (`render(0)`) with no loop or listeners. Props: `count`/`amplitude`/`accent` (an `"r,g,b"` string). Honest ambient art, `aria-hidden`. Used as the radial-masked backdrop of the redesigned **404** page and as a full-width **Playground** experiment ("Thread field").
