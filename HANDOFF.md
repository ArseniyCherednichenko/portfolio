# portfolio — Session Handoff

> Arseniy Cherednichenko's personal portfolio site. Living context for a fresh
> Claude session (or the daily routine). Goal: grow this into a polished,
> motion-led, professional portfolio over time, applying the craft from Guided.

## Stack
React 18 + Vite + TypeScript (strict) + **Tailwind v4** (via `@tailwindcss/vite`, classes auto-detected, no content config) + **Framer Motion** for animation + **react-router-dom v6** for client-side routing. Dark theme, lime accent `#DCF87C`.

## Architecture (multi-page)
- `src/main.tsx` wraps `<App/>` in `<BrowserRouter>`.
- `src/App.tsx` = route table only (`<Routes>`).
- `src/components/Layout.tsx` = persistent shell (Aurora + Nav + footer) with `<AnimatePresence>` page transitions (keyed on pathname), a `ScrollManager` that scrolls to `#hash` targets or resets to top on navigation, a site-wide `ScrollProgress` reading bar (top edge) and a `BackToTop` floating control.
- **Scroll orientation system**: `ScrollProgress` (springed lime reading bar pinned to the viewport top, `useScroll`+`useSpring`), `BackToTop` (floating bottom-right control that springs in past one screenful, smooth-scrolls home), both site-wide in `Layout`; `SectionNav` (`src/components/SectionNav.tsx`) is a `lg:`-only right rail used on Home — an `IntersectionObserver` scroll-spy over the `intro/about/work/toolkit/contact` section ids that lights the active marker (spring `layoutId` ring), labels reveal on hover, click glides to the section. All reduced-motion aware.
- Pages live in `src/pages/`: `Home.tsx` (hero/about teaser/range/work/toolkit/contact + project modal), `About.tsx` (intro, Now block, How-I-work principles, Path timeline, CTA), `Work.tsx` (`/work` index — animated ledger of all projects), `Playground.tsx` (live motion experiments gallery + full-width interactive `DotGrid` field), `WorkDetail.tsx` (`/work/:slug` project case study), `Contact.tsx` (`/contact` — deep-linkable contact page), `NotFound.tsx` (404).
- Routes: `/` Home, `/about` About, `/work` Work index, `/work/:slug` WorkDetail, `/toolkit` Toolkit, `/playground` Playground, `/contact` Contact, `*` NotFound. Work, About, Toolkit, and Playground are real page links in Nav (NavLink active styling); Contact is reachable via the command palette Pages group, the contact dialog's "Contact page ->" link, and direct/deep links. The Home page still has a `#work` section with the project modal; its header has an "All work ->" link to `/work`.
- **Work index** (`Work.tsx`): intro header (Eyebrow/GradientText + an `AnimatedCounter` of `CASE_STUDIES.length` shipped), then a row-based "ledger" mapping `PROJECTS` to `WorkRow`s. Real rows are `<Link>`s to `/work/:slug` with hover micro-interactions (growing lime left-edge, title nudge, sliding arrow), number/role/year/stack; the `soon` placeholder renders a calm non-clickable row. Staggered scroll reveal, reduced-motion aware. Closing CTA. The command palette's Work entry and WorkDetail's "All work" back-links point here.
- **Toolkit page** (`Toolkit.tsx`): deep-linkable `/toolkit`. Intro header (Eyebrow/GradientText "What I build with." + an `AnimatedCounter` of `TOOL_COUNT`), then the tools grouped by purpose (Languages, Interface, Backend and data, Craft and tooling) from `src/data/toolkit.ts`. Each group is a two-column row: a `md:sticky` label + blurb on the left, a 2-up grid of `SpotlightCard` tool cards on the right (name nudges + dot lights lime on hover, with an honest one-line note per tool). Staggered scroll reveal, reduced-motion aware. Closes with an honest "tools are just tools" note and a CTA into `/work` + `/playground`. The Home `#toolkit` marquee now has a "Full toolkit ->" link here; also in Nav and the command palette Pages group.
- **Toolkit data** (`src/data/toolkit.ts`): `TOOLKIT` (`ToolGroup[]` of `{ label, blurb, tools: {name, note}[] }`) is the single source of truth for what Arseniy builds with. `SKILLS` (flat name list for the Home marquee) and `TOOL_COUNT` derive from it; `projects.ts` re-exports `SKILLS` from here. Keep notes honest — real, in-use tools only.
- Shared data in `src/data/projects.ts` (`PROJECTS`, `SKILLS`, `Project` type with `slug`/`role`/`sections`/`highlights` case-study fields; `getProject(slug)` and `CASE_STUDIES` helpers). About-page content (Now/Principles/Path) lives inline in `About.tsx` — keep it honest and current.
- **Project detail pages** (`WorkDetail.tsx`): title block (role/year/GradientText), Visit/Source CTAs, stack chips, a 3-up highlights grid, a two-column narrative from `sections`, and a "Next project" SpotlightCard cycling `CASE_STUDIES`. Unknown/placeholder slugs render an inline not-found. The Home project modal's primary CTA and the command palette's Projects entries both route here.
- `vercel.json` rewrites all paths to `/index.html` so deep links resolve on static hosts.
- **Command palette** (`src/components/CommandPalette.tsx`): site-wide Cmd/Ctrl+K. `CommandPaletteProvider` wraps the shell in `Layout`, owns the global shortcut + open state, and exposes `useCommandPalette()` (Nav uses it for its search button). Fuzzy-searches Pages, Projects (derived from `PROJECTS`), and Actions (email, copy email, GitHub). Arrow-key nav, Enter to run, Esc/backdrop to close; reduced-motion aware, dialog + listbox a11y.
- **Contact dialog** (`src/components/ContactDialog.tsx`): site-wide "Get in touch" modal. `ContactProvider` wraps the shell in `Layout` (outside `CommandPaletteProvider`, so the palette can use it) and exposes `useContact()`. Reuses the shared `Modal`; renders the shared `ChannelList`; ends with a "Contact page ->" link into `/contact`. Opened from the Nav "Get in touch" button, the footer link, the command palette's "Get in touch" action, and Home's "Other ways to reach me" link. Reduced-motion aware.
- **ChannelList** (`src/components/ChannelList.tsx`): shared, honest list of reach-me rows (Email mailto, Copy email with confirmation, GitHub) driven by `src/data/contact.ts` (`CHANNELS`, `EMAIL`, `GITHUB_URL`). Owns the row markup, copy-to-clipboard behaviour, and channel icons — single source of truth used by both the Contact dialog and the `/contact` page so they never drift. Staggered reveal, reduced-motion aware.
- **Contact page** (`Contact.tsx`): deep-linkable `/contact`. Header (Eyebrow/GradientText + intro), a two-column block — left `SpotlightCard` with the shared `ChannelList`, right column with a **live Berlin clock** (`useBerlinTime`, updates each second via `Intl.DateTimeFormat` `Europe/Berlin`, with an honest "around / probably asleep" status dot that pings only when awake + motion allowed) and a "no forms" note — a 3-up "What I am open to" honest-interests grid, and a closing line linking to `/work` and `/playground`. No fabricated availability or response-time promises.
- **Keyboard navigation** (`src/components/Keyboard.tsx`): site-wide `KeyboardProvider` (mounted in `Layout`, wrapping `CommandPaletteProvider` so the palette can reach it) exposing `useShortcuts()` -> `{ openShortcuts }`. Two layers: (1) **"go to" chords** — tap `g`, then a page key from `GO_TARGETS` (h Home, w Work, a About, t Toolkit, p Playground, c Contact) to navigate; a `ChordHint` badge springs in bottom-center while armed and disarms after ~1.4s or on any other key. (2) A **shortcuts help dialog** opened with `?` (reuses the shared `Modal`), grouping "Go to" chords + "Command" keys (Cmd/Ctrl+K, ?, esc) as styled `kbd` rows with staggered reveal. Global handling bails when typing in a field/contenteditable, when meta/ctrl/alt is held, or when any `[aria-modal="true"]` surface is open (so it never fights the palette/contact/lightbox). `GO_TARGETS` is the single source of truth, reused by the help dialog and the Playground card. Reduced-motion aware. Surfaced via: a "Keyboard shortcuts" command-palette Action (hint `?`), a "Press ? for shortcuts" footer button (desktop), and a Playground experiment card.
- **Generative posters** (`src/components/ProjectPoster.tsx`): each project now has a deterministic, animated "poster" — drifting blurred colour fields (lime always leads, two seeded companions from an on-brand palette), a faint masked grid, a dark vignette, and the project name set large. The slug seeds a stable `mulberry32` RNG so the same project always renders the same composition. It is honest brand art, not a screenshot, and never claims to show real UI. Used as: the expandable hero on `WorkDetail` (wrapped in a `Lightbox`), a `md+` thumbnail on each real `Work` index row, and the banner inside the Home project modal.
- **Lightbox** (`src/components/Lightbox.tsx`): reusable focused fullscreen viewer — darkens the page, springs content up to fill the frame, optional caption, closes on Escape / backdrop / close control, locks body scroll, reduced-motion aware (cross-fades instead of scaling). Currently expands the `WorkDetail` poster; ready for any future visual media.
- **Cursor** (`src/components/Cursor.tsx`): site-wide custom cursor mounted in `Layout`. A tight lime dot springs hard toward the pointer while a softer ring lags behind; over interactive elements (`a, button, [role=button], input, textarea, select, label, summary, [data-cursor]`) the ring swells and fills faintly lime and the dot fades out, and it shrinks on press. Activates ONLY on `(pointer: fine)` with motion allowed — touch and reduced-motion users keep the native cursor (it returns `null` otherwise). When active it adds `body.cursor-none` (`index.css`) to hide the native cursor; layers fade out when the pointer leaves the window. Add `data-cursor` to any element to trigger the hover state. Featured as a Playground experiment card.
- **ScrollVelocity** (`src/components/ScrollVelocity.tsx`): React Bits-style scroll-velocity text band. Takes `rows` (`{ text, baseVelocity?, className? }[]`); each row renders 4 copies of its text in a `motion.div` whose `x` is a `wrap`ped offset driven by an `useAnimationFrame` loop — a constant base drift that speeds up and flips direction with the page's scroll velocity (`useScroll`->`useVelocity`->`useSpring`->`useTransform`). Per-row `className` lets rows differ (used to pair a filled white row with a lime `-webkit-text-stroke` outlined row). Under reduced-motion it renders one static, non-clipped line per row. Used full-bleed on Home (statement band between Work and Toolkit, masked with edge gradients) and as a Playground experiment.
- **DotGrid** (`src/components/DotGrid.tsx`): self-contained interactive canvas dot grid. Builds an evenly-spaced grid (configurable `gap`/`dotSize`/`proximity`/`push`), each dot a tiny spring pushed away from the pointer and pulled back to its origin, brightening toward lime within `proximity`. DPR-aware, `ResizeObserver` rebuilds on resize, RAF loop cleaned up on unmount; under reduced-motion it draws a single static evenly-lit grid (no loop). Used as the full-width "Interactive field" showcase on the Playground; reusable as an ambient layer anywhere.
- **CardStack** (`src/components/CardStack.tsx`): React Bits-style auto-advancing 3D card deck. Takes `cards` (`StackCard[]` of `{ tag, title, body }`) + optional `interval`. The front card recedes to the back on a timer; cards behind peek above it (offset/scale/tilt for depth, computed by `pose(pos)`). Advances on click, on a progress-dot jump, or the timer; pauses on hover/focus; only the front card is focusable/interactive (`aria-hidden` + `pointer-events` gated). Under reduced motion it renders a calm static, fully-readable list with no timer. Used on Home's new **Range** section (`#range`, between About and Work) to frame Arseniy as a multi-disciplinary builder — frontend, native iOS, backend/data, applied AI, motion/design — actively de-centering Guided; also a Playground experiment. `Home.tsx`'s `SECTIONS`/`SectionNav` includes the `range` id.
- **DecryptedText** (`src/components/DecryptedText.tsx`): text that resolves from random glyphs into the final string left-to-right. Plays once on view (`useInView`) and replays on hover; honest `aria-label` carries the real text. Under reduced-motion it renders the final string immediately with no scrambling. Used in the Playground "Decrypt-on-view text" experiment.
- **Playground** (`src/pages/Playground.tsx`): header + a 2-col grid of bite-size experiments (TiltCard, SpotlightCard, AnimatedCounter, MagneticButton, gradient+rotating type, DecryptedText, keyboard navigation, stateful micro-interactions) followed by a full-width `ScrollVelocity` band, the full-width `DotGrid` interactive field, and a closing CTA. Add new experiments here as the components library grows.
- **SEO / metadata**: `index.html` carries full base meta — title, description, author, `theme-color`, favicon (`/favicon.svg`, lime "A" + orbit-dot mark), `apple-touch-icon`, web manifest (`/public/site.webmanifest`), and Open Graph + Twitter `summary_large_image` tags. The OG/Twitter `image` is `/og.png` (relative, resolves against whatever host serves the site — no domain is hardcoded since none is set yet). The share image is **generated** by `scripts/generate-og.mjs` (zero-dep: hand-rolled PNG via Node's built-in `zlib`, no native libs) into `public/og.png` (1200x630 brand art — lime-led colour fields, faint grid, an orbit ring + glowing dot echoing `HeroOrbit`, vignette; no rasterised text). Re-run with `npm run og`. Honest brand art, not a screenshot.
- **Per-route head** (`src/components/Seo.tsx`): `<Seo title? description? />` renders nothing; in an effect it sets `document.title` (page title suffixed with the site name, or the default for Home), `description`, and keeps `og:title/description/url` + `twitter:title/description` in sync (`og:url` = current `window.location.href`, host-agnostic). Restores site defaults on unmount. Wired into **every** page with honest, distinct copy; `WorkDetail` uses the project's `title`/`blurb` dynamically (and a not-found variant). Before this, all routes shared one static title.
